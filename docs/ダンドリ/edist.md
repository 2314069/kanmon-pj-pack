## フェーズ2: 事前記事の企画・配信

### 記事の3フェーズ

| フェーズ | 目的 | 例 |
|---------|------|---|
| **事前記事** | 「面白いことが始まりそう」という空気感を伝える | タイトル決定、インターブッキング、司会紹介 |
| 当日記事 | リアルタイムで感門の熱を伝える | 各コーナーレポート、ダスト記事 |
| 事後記事 | 振り返りと記録 | まとめ記事、P-1記事 |

> **「前」記事を増やそう、と意識する**

### 事前記事の企画例

| 日付 | 記事内容 |
|-----|---------|
| 8/20 | 感門之盟タイトル決定 |
| 8/21 | Ｐ-１グランプリへの道 vol.1 |
| 8/23 | インターブッキング |
| 8/27 | 発掘ブックウェア |
| 8/28 | アフ感 |
| 8/31 | Ｐ-１グランプリへの道 vol.2 |
| 9/2 | 冠界式、司会紹介、JUSTライター決起、ジュリー |
| 9/5 | 本楼リハーサル |

> 発掘記事の番外編も加えれば、**11本の記事**が公開された

## フェーズ3: 当日の準備

### 外部記者連携MTG

- [ ] JUSTチーム・渦チームとの連携確認
- [ ] 2回目のミーティングで外部記者をまじえる

### 当日の記事担当決め

| 種類 | 特徴 | 担当の決め方 |
|-----|------|-------------|
| アーカイブ系 | 必ず残したいプログラム記事 | 事前に担当を決める |
| ダスト系 | 記者の視点で切り取る記事 | フリー（各自の判断） |

**分担のコツ:**
- 基本、講座師範の絶必担当記事は1本であとはフリー
- この分担がうまくいくと、ダスト系が増える
- **当日公開20本という成果も可能**

### 場所、配信の進め方の確認

- [ ] 当日の記者の居場所
- [ ] 記事配信の進め方
- [ ] 次第を確認しながらの記事担当決め

## フェーズ4: 当日

### 記者の動き方

1. **まず素材集めを始める**
   - 記事を書くには素材がないとだめ
   - とりあえずメモを取る
   - 写真も適当に撮る

2. **担当コーナーに集中**
   - ミッションがないと、書けないまま終わる可能性も

3. **本楼以外も見る**
   - ずっと本楼に詰めていると、2階の様子を見逃す
   - 場所ごとにゆるく分担があってもよい

### 投稿前のチェック

- [ ] 最新の遊刊エディストをチェック
- [ ] 情報がダブっていないか確認
- [ ] タイポチェック

## 師範だからこそ書ける記事

> 「師範だからこそ書ける記事」がたくさんある

- 今後も感門エディストチームが発足できれば、その強みを活かせる
- 講座の内側を知っているからこその視点
- 講座間で記事のアイデアを出し合う
- 「講座しか知り得ないニュース」を共有できる

## 写真撮影の課題と改善案

### 現状の課題

- 記者が撮影もし、記事も書くのは、タスクとして非常に多い
- 「面白い場面が切り取れた」も記事を書く際に大事な視点

### 改善案

| 案 | メリット | デメリット |
|---|---------|-----------|
| 写真専門のロールを置く | クオリティ向上、記者の負担軽減 | 厳密に分けると柔軟性が減る |
| 別の記者に撮影をお願い | 柔軟な対応 | 受け渡しの手間 |
| BOXにバンバン上げてもらう | 共有しやすい | 整理が必要 |

## JUST記者へのガイド

### 初めての人へのアドバイス

1. **まずは素材集め**から始める
2. **メモを取る**習慣をつける
3. **写真は多めに**撮っておく
4. **読み手を1人決めて**書く（例：感門に参加できない期外の師範代）
5. **最初の1本**に集中する

### 事前に準備しておくべき情報

- P-1登壇する学衆さんのお名前
- 教室名
- **事前に準備しておけばよかった、と後悔しがち**

> 詳細は [JUST記者ガイド](../04_ナレッジ/エディスト/JUST記者ガイド.md) を参照
